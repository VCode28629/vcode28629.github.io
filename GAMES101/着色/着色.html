着色Shading：对不同的物体应用不同材质的过程

基础着色模型：Blinn-Phong Reflectance Model

  • 高光
  • 漫反射
  • 间接光照（路径追踪部分介绍）

考虑每个Shading point

  • 视线方向$\hat v$
  • 表面法线方向$\hat n$
  • （很多条）光照方向$\hat l$
  • 表面参数（颜色等）

不考虑阴影

漫反射

$L_d=k_d\frac I{r^2}\max(0,\hat n\cdot\hat l)$

$k_d$：diffuse coefficient(color)，越大物体越亮

$\frac I{r^2}$：到达shading point的能量

镜面反射

高光：Phong：观察方向和镜面反射方向足够接近的时候

Blinn-Phong：法线方向$\hat n$和半程向量$\hat h$方向接近

半程向量$\hat h=\mathrm{bisector}(\hat v,\hat l)=\frac{\hat v+\hat l}{|\hat v+\hat l|}$

$L_s=k_s\frac I{r^2}\max(0,\hat n\cdot\hat h)^p$

p越小，高光条越小。通常是100~200。

环境光照

$L_a=k_aI_a$

合并

$L=L_a+L_d+L_s=k_aI_a+k_d\frac I{r^2}\max(0,\hat n\cdot\hat l)+k_s\frac I{r^2}\max(0, \hat n\cdot\hat h)^p$

着色频率

Shade each triangle(flat shading)

  • 每个三角形面都是平的，法线是同应该 向量
  • 对平滑表面不适用

Shade each vertex(gouraud shading)

求每个顶点，然后对平面插值

Shade each pixel(Phong shading)

  • 对每个三角形插值法线向量
  • 对每个像素计算shading
  • 不是Blinn-Phong Reflectance Model

如何求一个顶点的法线

相邻的面的法线的平均

$\hat N_v=\frac{\sum_i\hat N_i}{|\sum_i\hat N_i|}$

更好的：以相邻面的面积加权平均

如何用顶点法线求面上的法线

纹理

纹理就是一张图，空间中的三角形映射到平面上。

纹理上的坐标系通常用(u,v)定义。(u,v)的范围通常是$[0,1]^2$

不同的位置可以映射到纹理的一个地方。

可拼接的纹理tiled

纹理可以表示环境光

如何在三角形内部插值

重心坐标Barycentric Coordinates

重心坐标是定义在三角形上的。

三角形$ABC$平面上任何一个点的坐标都可以表示成三角形顶点坐标$ABC$的线性组合$\alpha A+\beta B+\gamma C$，$\alpha+\beta+\gamma=1$。所以用$(\alpha,\beta,\gamma)$来表示三角形平面内一个点。由于$\alpha+\beta+\gamma=1$，所以知道两个就能知道第三个。当$\alpha,\beta,\gamma\ge0$，则点一定在三角形内。

重心坐标可以通过面积比得出：将AP、BP、CP、连线。点$P$的重心坐标为$\left(\frac{S_{PBC}}{S},\frac{S_{PAC}}{S},\frac{S_{PCB}}{S}\right)$

插值时用重心坐标加权平均属性。

投影不能保证重心坐标一致。所以三维空间中的属性要在三维空间中插值。

多个pixel映射到同一个texel上

纹理上的像素：texel，纹理元素，纹素

解决方案：双线性插值

更好的解决方案：bicubic，选周围16个来插值。

纹理太大

远处摩尔纹，近处锯齿

原因：不同的像素在纹理上覆盖区域不同

解决方案1：超采样——可以，但是太慢了

解决方案2：不采样
